# v0.2 Chunk 3 — SOL + SPL Token Detection

> Read `v0.2-overview.md` first. Read `v0.2-eth.md` because the Solana chunk follows the same pattern as ETH and reuses the same UI flow and metadata shape.

## Goal

Let a user add a Solana address as an asset row. Track SOL balance AND auto-detect significant SPL token balances. Sync every 30 minutes via Helius RPC.

## Scope

- Single watch-only Solana address support (`base58` 32-byte pubkey)
- SOL balance + auto-detected SPL token balances
- Per-token USD via CoinGecko (token-by-contract endpoint also works for Solana via `solana` chain id)
- Stablecoin auto-classification (USDC-SPL, USDT-SPL)
- Cron entry every 30 min
- Address validation (base58, 32-44 char range, valid base58 charset)
- Free Helius API key required (env var `HELIUS_API_KEY`)

## Out of scope

- Staked SOL accounts (stake program — separate provider call, defer)
- NFTs (Helius has cheap NFT discovery but huge response sizes — defer)
- Liquid staking tokens (mSOL, jitoSOL, bSOL — they ARE detected as SPL tokens, just don't try to "unwrap" them to SOL)
- Drift / Mango / Marinade DeFi positions (Zerion handles these in a later chunk)
- Compressed NFTs / state compression
- Token-2022 program (newer SPL standard) — should detect them via Helius getAsset, no special-casing needed
- Per-token historical snapshots

## Decision: Helius DAS API vs RPC

Helius offers:
1. **Standard Solana RPC** at `https://mainnet.helius-rpc.com/?api-key=...` — call `getBalance` for SOL, `getTokenAccountsByOwner` for SPL
2. **DAS (Digital Asset Standard) API** — `getAssetsByOwner` returns SOL + tokens + NFTs in one call

**Recommendation:** Use the DAS API. One call instead of N+1, returns prices for known tokens, includes USD values. Filter `interface=FungibleToken` to drop NFTs.

If DAS feels heavy or the response is too noisy, fall back to standard RPC `getTokenAccountsByOwner` + a separate price lookup loop.

## Data layer

Same as ETH chunk: store everything in `assets.metadata` (Option A flat) or use a `holdings` table (Option B nested). Use the same shape as the ETH metadata so the detail panel can render both with one component.

## Provider implementation: `src/lib/providers/helius.ts`

```ts
export interface SolAddressInfo {
  address: string;
  solBalance: bigint;     // lamports
  solBalanceFormatted: number;
  tokens: SolTokenBalance[];
}

export interface SolTokenBalance {
  mint: string;           // SPL token mint address
  symbol: string;
  name: string;
  decimals: number;
  rawBalance: bigint;
  balance: number;
  priceUsd?: number;      // from Helius DAS or CoinGecko
}

export async function getSolBalance(address: string): Promise<SolAddressInfo>;
```

Helius DAS endpoint:
```
POST https://mainnet.helius-rpc.com/?api-key={HELIUS_API_KEY}
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "getAssetsByOwner",
  "params": {
    "ownerAddress": "<address>",
    "page": 1,
    "limit": 100,
    "displayOptions": { "showFungible": true, "showNativeBalance": true }
  }
}
```

Parse the response:
- `result.nativeBalance.lamports` → SOL balance (and `priceInfo.totalPrice` for USD)
- `result.items[]` filtered to `interface === "FungibleToken"` → SPL tokens with `token_info.balance`, `token_info.symbol`, `token_info.decimals`, `token_info.price_info.price_per_token`

This is much simpler than ETH because Helius pre-prices tokens.

## Failure modes

| Mode | Handling |
|---|---|
| HELIUS_API_KEY missing | Startup-checks warn; cron skips |
| Helius rate limit (free tier: 100k credits/month, ~3 req/sec) | Use rate-limit-cache, batch addresses with sleep |
| DAS returns price=0 for a token | Skip pricing, still include the token in the list |
| Large wallet with 200+ tokens (airdrops) | Filter to non-zero-USD tokens, top 50 by value |
| Compressed NFTs in response | Filter them out via `interface !== "V1_NFT"` |
