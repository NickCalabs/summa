# v0.2 Chunk 2 — ETH + ERC-20 Token Detection

> Read `v0.2-overview.md` first. Read `v0.2-btc.md` because the BTC handoff establishes patterns this chunk extends.

## Goal

Let a user add an Ethereum address as an asset row. Track ETH balance AND auto-detect significant ERC-20 token balances. Each token gets its own value rolled up into a single wallet asset (or as nested holdings — see decision below). Sync every 30 minutes via Etherscan.

## Scope (this PR)

- Single watch-only ETH address support
- Auto-detect ERC-20 tokens with non-zero balance
- Per-token USD value via CoinGecko (we already have many tokens cached) + Etherscan token info as fallback
- Stablecoin auto-classification (USDC, USDT, DAI → cash equivalent flag set on the holding)
- Cron entry: every 30 minutes, refresh all `chain=eth` wallets
- Address validation (`0x` + 40 hex chars + checksum check)
- Rate-limit handling (Etherscan free tier: 5 calls/sec, 100k calls/day)

## Out of scope

- ENS reverse-resolution (use the raw address; no `vitalik.eth` lookup)
- NFT inventory (Etherscan does this but it explodes the surface area; v0.5 alt-assets)
- DeFi positions in protocols (Aave, Compound, etc.) — covered by Zerion in a later v0.2 chunk
- Multi-chain EVM (Polygon, Arbitrum, Base, Optimism) — same architecture but separate addresses; follow-up PR
- Per-token historical snapshots (see overview's "out of scope")
- Gas fee tracking
- Internal transactions

## Key decision: how do tokens render in the UI?

**Option A — Flat (recommended for first PR):** the ETH wallet is one asset row. Its `currentValue` is the sum of ETH + all ERC-20 token values. Token breakdown is shown only in the slide-out detail panel as a list. The asset table shows the wallet's name, total value, and a count badge ("3 tokens"). Simpler, fits the existing schema.

**Option B — Nested (more complex):** the ETH wallet is one parent asset row. Each ERC-20 token is a child holding. Requires a new `holdings` table with `parent_asset_id`. The asset table shows expandable rows. This is what Kubera does. **Defer to a follow-up PR** unless you have time and a clear plan.

If you go with Option A, you'll store the token list in `assets.metadata`:

```ts
metadata: {
  tokens: [
    { symbol: "USDC", contractAddress: "0xa0b8...", balance: "1500.00", priceUsd: 1.0, valueUsd: 1500.0, isStablecoin: true },
    { symbol: "LDO", contractAddress: "0x5a98...", balance: "320.5", priceUsd: 1.85, valueUsd: 592.92, isStablecoin: false },
  ],
  ethBalance: "1.85",
  ethPriceUsd: 3450.0,
  totalUsd: 8474.42,
  lastSync: "2026-04-09T03:30:00Z"
}
```

The detail panel reads this and displays a sub-table.

## Data layer

If Option A: schema is already prepared. No new tables.

If Option B: new `holdings` table with `parent_asset_id` foreign key, `symbol`, `contract_address`, `balance`, `price_usd`, `value_usd`, `metadata` JSONB. Generate migration via `pnpm db:generate`. Restart dev server after applying.

## Provider implementation: `src/lib/providers/etherscan.ts`

Etherscan v2 API (the multichain endpoint) is preferred over the legacy single-chain endpoint. Free tier API key required. Document the env var: `ETHERSCAN_API_KEY`.

```ts
export interface EthAddressInfo {
  address: string;
  ethBalance: bigint;     // wei
  ethBalanceFormatted: number;  // for convenience
  tokens: EthTokenBalance[];
}

export interface EthTokenBalance {
  contractAddress: string;
  symbol: string;
  name: string;
  decimals: number;
  rawBalance: bigint;
  balance: number;        // formatted using decimals
}

export async function getEthBalance(address: string): Promise<EthAddressInfo>;
export async function getEthBalanceBatch(addresses: string[]): Promise<Map<string, EthAddressInfo>>;
```

Endpoints needed:
- `?module=account&action=balance&address=...` — ETH balance
- `?module=account&action=tokenlist&address=...` — list of all tokens the address has interacted with
- `?module=account&action=tokenbalance&contractaddress=...&address=...` — balance for a specific token

**Optimization:** Etherscan also has an `addresstokenbalance` endpoint that returns all token balances in one call. Use it if available; if not, batch the per-token calls with a 200ms sleep between.

**Filter:** Drop tokens with `balance === 0` and tokens with no symbol/decimals (spam tokens).

**Stablecoin detection:** Maintain a small allowlist of known stablecoin contract addresses (USDC, USDT, DAI, FRAX, LUSD, TUSD on mainnet). If a token's `contractAddress` is in the list, set `isStablecoin: true`. Don't try to be clever — just match the addresses.

## CoinGecko price lookup for ERC-20 tokens

CoinGecko has a "token by contract" endpoint:
`https://api.coingecko.com/api/v3/coins/ethereum/contract/{contract_address}`

Use it sparingly (rate limit: 30 req/min on free tier). Cache aggressively in `rate-limit-cache.ts` — token prices don't need to refresh every cycle.

For tokens not on CoinGecko, set `priceUsd: 0` and `valueUsd: 0`. Don't error out.

## Cron

```ts
cron.schedule("*/30 * * * *", async () => {
  const wallets = await db.select().from(assets)
    .where(and(
      eq(assets.providerType, "wallet"),
      eq(assets.isArchived, false),
      sql`${assets.providerConfig}->>'chain' = 'eth'`
    ));

  if (wallets.length === 0) return;

  for (const wallet of wallets) {
    try {
      const info = await getEthBalance((wallet.providerConfig as any).address);
      const ethPriceUsd = await getCoinGeckoPrice("ethereum", "usd");
      const tokens = await Promise.all(info.tokens.map(async t => {
        const price = await getTokenPriceFromCoinGecko(t.contractAddress);
        return {
          symbol: t.symbol,
          contractAddress: t.contractAddress,
          balance: String(t.balance),
          priceUsd: price ?? 0,
          valueUsd: t.balance * (price ?? 0),
          isStablecoin: STABLECOIN_ADDRESSES.has(t.contractAddress.toLowerCase()),
        };
      }));
      const ethValueUsd = info.ethBalanceFormatted * ethPriceUsd;
      const tokenValueUsd = tokens.reduce((sum, t) => sum + t.valueUsd, 0);
      const totalUsd = ethValueUsd + tokenValueUsd;

      await db.update(assets).set({
        quantity: String(info.ethBalanceFormatted),
        currentPrice: String(ethPriceUsd),
        currentValue: String(totalUsd),
        metadata: { ...wallet.metadata, ethBalance: String(info.ethBalanceFormatted), ethPriceUsd, tokens, totalUsd, lastSync: new Date().toISOString() },
        lastSyncedAt: new Date(),
        updatedAt: new Date(),
      }).where(eq(assets.id, wallet.id));
    } catch (err) {
      console.error(`[cron] ETH wallet ${wallet.id} refresh failed:`, err);
    }
  }

  console.log(`[cron] ETH wallet refresh: ${wallets.length} processed`);
});
```

## Failure modes

| Mode | Handling |
|---|---|
| Etherscan API key missing | Startup-checks should warn (similar to ENCRYPTION_KEY pattern). Cron silently skips. |
| Etherscan rate limit (1 req/sec free tier) | Use rate-limit-cache, queue calls, never burst. |
| Token contract has no decimals | Treat as 18 (default ERC-20) and log a warning. |
| CoinGecko 404 for token | priceUsd=0, valueUsd=0, still include in token list. |
| Address has 1000+ tokens (spam) | Filter to top 50 by USD value before storing in metadata. |
| Etherscan returns "max rate limit reached" | Backoff 60s, retry on next cycle, don't tight-loop. |
